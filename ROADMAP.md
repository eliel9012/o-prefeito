# Roadmap

## Fase 0 — Preparação do fork

- Fazer fork de `amilich/isometric-city`.
- Clonar localmente.
- Confirmar licença MIT.
- Rodar `npm install`.
- Rodar `npm run build`.
- Criar branch `prefeito-brasil-mvp`.
- Adicionar estes arquivos `.md` ao repositório.

## Fase 1 — Rebranding mínimo

- Nome do app: `O Prefeito`.
- Subtítulo: `Simulador de Cidade Brasileira`.
- Textos principais em PT-BR.
- Cores iniciais mais brasileiras, sem caricatura.
- Tela inicial simples.

## Fase 2 — macOS/Tauri

- Ajustar Next.js para export estático.
- Adicionar Tauri.
- Configurar `frontendDist` para `../out`.
- Criar app icon provisório.
- Gerar `.app` no Mac.
- Validar save/load.

## Fase 3 — Sistemas brasileiros mínimos

- Orçamento municipal.
- Receita: IPTU, ISS, repasses.
- Despesa: saúde, educação, saneamento, segurança, obras.
- Popularidade do prefeito.
- Eventos mensais simples.

## Fase 4 — Eventos brasileiros

- Enchente.
- Buracos nas ruas.
- Obra parada.
- Licitação contestada.
- Ministério Público recomenda providências.
- Câmara Municipal pressiona orçamento.
- Imprensa local afeta popularidade.

## Fase 5 — Identidade visual brasileira

- Fachadas comerciais brasileiras.
- Casas populares e prédios residenciais brasileiros.
- Prefeitura, praça central, terminal de ônibus.
- Postes, fiação, calçadas irregulares, árvores e sinalização.
- Assets gerados com Higgsfield como concept art e, se viável, adaptados para o jogo.

## Fase 6 — MVP jogável

- Loop de jogo claro.
- Objetivos iniciais.
- Tutorial curto.
- Balanço mensal.
- Eventos com escolhas simples.
- Build macOS funcional.

## Fora do escopo inicial

- Multiplayer.
- Simulação de trânsito realista demais.
- Política partidária real.
- Marcas reais.
- Pessoas públicas reais.
- Sistema jurídico profundo demais.
- Importação de mapas reais.
